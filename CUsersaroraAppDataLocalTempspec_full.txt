1. Project Overview
Brief: A single-page (long-scroll) website with five auxiliary legal/compliance pages. Lead-capture
optimised. Built fast, designed to last 18 months before a v2 rebuild after the first cohort generates real
outcomes data.
Primary goals (in priority order):
1. Capture student applications for the first cohort (Track One)

2. Capture industry partnership interest (hire, volunteer, support)
3. Capture donor / CSR enquiries
4. Establish credibility through transparent governance disclosure
5. Provide press / media a starting point
Out of scope for v1: Online donation processing, login/portal, content management for students, blog or
news section, multi-language. These come in v2 once the first cohort is live and there’s actual content to
publish.
2. Technical Stack Recommendation
The site is small, content-heavy, lead-capture focused. Optimise for speed of build and ease of edits.
Recommended stack:
Framework: Next.js 14+ with App Router (or Astro if developer prefers — both work for a static-leaning
site)
Styling: Tailwind CSS
Forms: Server actions writing to a database (Supabase or Convex) + email notifications via Resend or
Postmark
Hosting: Vercel (free tier sufficient for v1 traffic levels)
Domain & DNS: Cloudflare for DNS + edge caching
Email: Forms write to database AND send a notification email to a Foundation inbox (e.g.
hello@mcbeefoundation.in )
Forms backup: Every submission also goes to a Google Sheet via Zapier or n8n as a redundancy
Alternative simpler stack (if developer is non-technical):
Webflow with Forms + Memberstack — drag-and-drop, slightly more expensive (~₹2,500/month) but no
developer needed for content updates
Avoid: WordPress (overkill, security overhead), Wix (limits on form customisation), pure HTML/CSS (forms
become a hassle).
No browser storage for form data; use server-side persistence only.
